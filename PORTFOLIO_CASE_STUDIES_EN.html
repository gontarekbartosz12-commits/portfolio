1 · Polymarket Trading Bots

Python · Polymarket Gamma & CLOB API · Polygon/Web3 (py-clob-client) · Simmer API · Linux VPS + cron · Telegram

The problem

Prediction markets misprice fast-moving events. I wanted to find out, with real code and real data, whether a retail-sized account can extract an edge — and which strategies survive contact with the market.

What I built

  • Copy-trader — tracks top on-chain wallets, detects new positions via the Polymarket data API and mirrors them at a fixed % of balance with exposure caps.
  • AI-divergence scanner — trades only when an independent AI fair-value estimate diverges from the live price by ≥10 pp.
  • BeastBot view-count model — quant model for YouTube-views markets (project 2).
  • Limitless LP bot — automated liquidity provision on Base chain with its own backtester + monitor.

Engineering

  • Position diffing — snapshots holdings per wallet and trades only on the delta, never re-copying stale positions.
  • Risk layer — copy %, min/max trade size, daily-trade and drawdown caps enforced before any order.
  • Resilience — every call wrapped in timeouts + try/except; the loop self-heals so a flaky API never kills it.
Honest result: run in paper/$SIM mode first. The most valuable outcome was negative — by auditing ~3,380 on-chain trades I proved naïve copy-trading is unprofitable at retail scale (≈ −$222/week on $300). That killed a bad strategy before risking real money and steered the project to model-driven edges.

Demonstrates: real API + Web3 integration, risk management, and the discipline to falsify my own ideas with data.

2 · BeastBot — quant model for YouTube-views markets

Python 3.10+ · SciPy (sigmoid fit) · YouTube Data API v3 · Polymarket Gamma · Normal-CDF pricing · multi-agent consensus

The edge

Early view data follows a sigmoid growth curve. I fit it to project the saturation view count, convert to a bucket probability with a Normal CDF, and trade only when the market diverges from my model by ≥8 pp.

sources/   → live views (YouTube API) + scraper fallback
model/     → growth_curves (sigmoid) · historical baseline · bootstrap CI
hermes/    → 3 agents (api / curve / historical) → weighted consensus
pricer.py  → Normal CDF → bucket probabilities
trader.py  → main loop   ·   monitor.py → exit logic
  • Calibrated uncertainty — sigma widens for longer horizons (grounded in view-growth literature).
  • Consensus scoring — Hermes blends three signals with explicit weights; skips low-confidence markets.
  • Secrets done right — all keys via os.getenv(), nothing hardcoded.

Demonstrates: applied statistics/ML, clean modular architecture, research → deployable risk-bounded system.
